# Setup Guide

## 1. Folder structure

```
siva-progy/                      ← this must be a repo named EXACTLY after your username
├── README.md                    ← the profile itself
├── SETUP.md                     ← this file (optional, delete once you're set up)
├── assets/
│   ├── banner.svg                ← animated terminal/candlestick banner
│   ├── divider.svg                ← candlestick section divider (reused)
│   ├── ticker.svg                  ← scrolling skills ticker (signature element)
│   └── snake.svg / snake-dark.svg  ← auto-generated by the workflow, don't create manually
└── .github/
    └── workflows/
        └── snake.yml             ← generates the contribution snake daily
```

## 2. One-time setup steps

1. Create a new **public** repository on GitHub named **exactly** `siva-progy` (your username). GitHub auto-detects this as your profile repo.
2. Upload this whole folder's contents to that repo (README.md, SETUP.md, assets/, .github/).
3. Go to the repo's **Settings → Actions → General** and make sure "Read and write permissions" is enabled for the `GITHUB_TOKEN` (needed for the snake workflow to push to the `output` branch).
4. Go to the **Actions** tab and manually run "Generate Contribution Snake" once (don't wait for the daily cron). This creates the `output` branch with `snake.svg`.
5. Refresh your profile page — the banner, ticker, stats, streak, trophies, and snake should all render.

## 3. What's using an external service vs. custom-built

| Element | Source |
|---|---|
| Banner | custom SVG (`assets/banner.svg`) — yours to edit |
| Ticker tape | custom SVG (`assets/ticker.svg`) — yours to edit |
| Divider | custom SVG (`assets/divider.svg`) — yours to edit |
| Typing intro | `readme-typing-svg` (external, free, no signup) |
| Stats / top languages / streak / trophies | `github-readme-stats` + `github-profile-trophy` (external, free) |
| Contribution snake | `Platane/snk` GitHub Action (runs in your own repo) |
| Profile view counter | `komarev.com/ghpvc` (external, free) |

None of these require API keys or sign-ups — they're all free public services widely used for GitHub profiles.

## 4. Easy customizations later

- **Swap in real projects:** replace the 3 placeholder cards in the "Featured Projects" section — update the title, description, tech badges, status, and `[repo link →](#)` URL.
- **Trim the tech stack:** delete any badge line for a tool you haven't actually used yet.
- **Recolor anything:** the palette is `#2ee6a6` (green), `#4fd1ff` (cyan), `#ff5c73` (red), `#10161f` (card bg) — find-and-replace these hex codes across the SVGs and badge URLs to reskin the whole profile at once.
- **Snake colors:** edit the `color_snake` / `color_dots` params in `snake.yml` if you want a different palette later.

## 5. If something doesn't render

- Stats/streak/trophies images sometimes take a few seconds on first load — refresh the page.
- The snake image will 404 until you've run the workflow at least once (step 4 above).
- SVGs with animation render fine directly on GitHub, but won't animate in some markdown previews (like VS Code's default preview) — that's normal, it'll animate on the live GitHub page.
